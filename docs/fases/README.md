## Sequencia Recomendada

Para uma entrega academica ou demonstracao funcional, priorize as fases 00 a 04. Elas fecham o nucleo do projeto: ambiente validado, demo local, fluxo escrow real, marketplace minimo e disputas/evidencias.

Para um MVP beta, avance tambem pelas fases 05 a 08. Elas adicionam reputacao verificavel, qualidade, staging em Base Sepolia e operacao controlada.

A fase 09 deve ser tratada como evolucao pos-validacao, guiada pelos aprendizados do beta.
